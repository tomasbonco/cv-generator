1.2. Riešenie úlohy

Mojou prvou úlohou bolo vytvoriť XML databázu, ktorá bude uchovávať všetky potrebné dáta užívateľov. Pôvodný plán bol použiť jeden XML súbor na uloženie dát všetkých užívateľov, neskôr sme sa však kvôli validácii a aj prehľadnosti rozhodli ukladať dáta jedného uživateľa do jedného XML súboru. To znamená, že pre každého užívateľa existuje separátny XML dokument. XML súbor je pomenovaný podľa vygenerovaného reťazca - hash (viď ďalšie kapitoly). Obsahuje všetky údaje konkrétneho životopisu vrátane hesla v šifrovanej podobe.

V ďalšej fáze som sa venoval tvorbe XML Schémy, ktorá slúži na zvalidovanie obsahu XML dokumentu. Schéma popisuje presnú šruktúru databázového súboru a je navrhnutá tak, aby vyhovovala základným požiadavkám, ktoré sú kladené na jednotlivé údaje životopisu, ale zároveň, aby ošetrila všetky neštandardné a nevalidné vstupy (vrátane násobného použitia údajov, ktoré sa môžu nachádzať v dokumente najviac jeden krát). Na overenie správnosti XML Schémy som použil unit testy, konkrétne som testoval existenciu / neexistenciu všetkých elementov / atribútov, nekorektné naplnenie hodnôt pre všetky elementy / atribúty, a tiež testy na prázdnosť elementov / atribútov.

Ďalšia časť mojej práce pozostávala z tvorby XSLT transformácie, ktorá má za úlohu previesť obsah XML dokumentu do formátu TeX. Ako štýl TeX dokumentu som zvolil triedu ECV – A fancy Curriculum Vitae class z webovej lokality http://ctan.org (The Comprehensive TeX Archive Network). Táto trieda poskytuje jednoduché prostredie pre vytvorenie tabuľkového životopisu, inšpirovaného európskym životopisom. Po úspešnej transformácii XML dokumentu na TeX dokument, nasleduje konverzia do PDF formátu (viď ďalšie kapitoly).
